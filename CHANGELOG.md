**2026-05-19 (later)** — **Context-management layer for long modding sessions: off-prompt move log + phase-boundary compaction + `query_history` meta-tool + session recovery + FE session_id persistence.** Motivation: a 30-min full-pipeline run is 10-15 turns at ~3 min/turn on `deepseek-v4-flash`, each turn re-sending the entire `_global_history` — token cost, latency, and eventually context exhaustion. The fix exploits ModPilot's existing "scene IS the memory" principle: prompt holds only enough to make the next decision; everything older lives off-prompt and is one tool call away.

New module `ModPilot/app/agent/history.py`:
- `MoveLog` — append-only JSON-Lines log at `~/.modpilot/sessions/{sid}/moves.jsonl`. Auto-injects `ts` and a monotonic `turn` counter (resumes from existing line count on reconstruction). Filters by `phase` / `kind` / `name` / `last_n`. Path-traversal session_ids rejected at construction. Survives malformed lines silently. Unicode payloads round-trip cleanly (lesson re: cp1252 already logged).
- `compact_phase_range(history, start, end, summary)` — pure function returning a new list with `history[start:end]` replaced by one assistant message prefixed with the `[compacted]` marker. Idempotent: re-compacting an already-compacted span is a no-op (prevents double-collapse in retry paths).
- `QUERY_HISTORY_TOOL_SCHEMA` — meta-tool definition the LLM sees. Optional filters: `phase`, `kind`, `name`, `last_n`. Server enforces `QUERY_HISTORY_DEFAULT_LAST_N=50` (when omitted/invalid) and `QUERY_HISTORY_MAX_LAST_N=1000` (clamp), so a misbehaving LLM cannot defeat compaction by yanking the whole log.

`AgentLoop` wiring (`ModPilot/app/agent/loop.py`):
- New optional `session_id` constructor param; existing tests pass nothing and the whole subsystem is a no-op (`_move_log is None`).
- `_log_move()` helper centralizes the None-check so all logging sites stay one-liners.
- `step()` logs `user` / `widget` (when message has `[CONFIRMED_CLASSIFICATIONS]` / `[CONFIRMED_MATERIAL_MAPPING]` prefix) / `error_choice` (when state == `ERROR_HANDLING`) at entry, `assistant` after dispatch returns. Classification happens BEFORE the user-message append because `_dispatch` may transition state mid-call.
- `_execute_tool_call` logs one `tool` move per call (phase tool, query tool, `sync_phase_state`, and `query_history` itself). On phase-advancing-tool success, additionally logs a `phase_advance` move (`phase` = just-completed, `to_phase` = next or null) and sets `_just_completed_phase = completed` alongside the existing Issue #15 `_phase_just_advanced` flag — the two flags coexist by design.
- `interrupt()` logs an `interrupt` move with current phase.
- New `_compact_completed_phase_if_pending()` runs at the top of `_run_react_turn`, BEFORE the round loop. When `_just_completed_phase` is set: scans the trailing in-history wrap-up assistant text, collapses `history[_phase_start_idx_global : len-1]` to one `[compacted] <wrap-up>` summary, advances `_phase_start_idx_global = old + 1`, clears the flag. Zero extra LLM cost — reuses the Issue #15 wrap-up text the loop already generates.
- New `_hydrate_from_move_log()` runs once at the end of `__init__`. Phase-granular recovery: counts `phase_advance` entries → `_phase_idx`; for each, finds the next logged `assistant` move → injects one `[compacted] <wrap-up>` summary into `_global_history` (fallback `"Phase X completed."` when the wrap-up was never logged); sets `_phase_start_idx_global` to `len(history)`. Mid-phase work is NOT replayed — scene-is-memory + `query_history` cover it. Faithfully reconstructing tool_use/tool_result blocks would require preserving exact `tool_use_id` strings (Anthropic API 400s on mismatch); flat compacted summaries align with the steady-state shape of compacted history anyway.
- `_build_tool_list` adds `QUERY_HISTORY_TOOL_SCHEMA` alongside `_SYNC_PHASE_TOOL_SCHEMA`. Both meta-tools are routed in `_execute_tool_call` (no `PhaseTool` subclass needed — they read off-Blender state).

System prompt (`ModPilot/app/agent/prompts.py`):
- New `_CONTEXT_MANAGEMENT_PROTOCOL` module-level constant injected into `build_system_prompt()` between `transition_protocol` and `_WIDGET_PROTOCOL`. Tells the LLM that older detail is compacted (with the `[compacted]` marker so it can recognize it in its own history), explains `query_history` with four concrete usage patterns, and gives a decision guide: live Blender query tools for current state, `query_history` for past decisions, narrow filters to keep `last_n` cheap.

Route layer (`ModPilot/app/main.py`):
- `_get_or_create_session` passes `session_id=session_id` to the `AgentLoop` constructor. One-line change.

Frontend (`ModPilot/frontend/src/lib/session.ts`):
- `getSessionId()` persists the 12-char hex id in `localStorage` (key `modpilot.session_id.v1`), validates the stored value against `/^[a-f0-9]{12}$/i` before trusting it, falls back to module-scope cache (legacy behavior) when storage throws. Without this, every page reload minted a fresh id, the backend always saw a new session, and the prior session's `moves.jsonl` was orphaned on disk — recovery never triggered in practice.
- New `resetSessionId()` export for a future user-driven "start fresh" affordance.

Tests:
- New `tests/unit/test_history.py` — 28 tests on the pure module: `MoveLog` path resolution + traversal rejection, append IO + ts/turn injection + Unicode round-trip, read filters + last_n + missing-file + malformed-line resilience, `compact_phase_range` edge cases including out-of-bounds and idempotent re-compaction, system-prompt injection assertions, schema shape.
- New `tests/unit/test_agent_loop_history.py` — 22 tests on the wiring: `MoveLog` plumbing on/off via `session_id`, per-turn user/widget/error_choice/assistant logging, tool-move + phase_advance logging, two-turn compaction with wrap-up text reuse, `_phase_start_idx_global` advancement, interrupt logging, query_history tool registration + execution + last_n cap (default/clamp/honored/fallback-on-invalid), six session-recovery scenarios (no log, single advance, multi-advance, advance-without-wrap-up, mid-phase no replay, turn counter resume).
- Full unit suite **526 passing** (was 476, +50). No existing tests modified — all additions land in two new test files per the AGENTS.md hard rule.

What this does NOT do (deferred): the on-disk `moves.jsonl` per-session has no GC — a long-lived install accumulates session dirs under `~/.modpilot/sessions/`. The Issue / 4-step recovery design called for an mtime sweep; not implemented in this pass. Move log size is bounded per session but not across sessions.

Behavioral note for live debugging: when a session is hydrated from disk, `_global_history` length is **not** equal to the move log line count — the log holds the unabridged record (one entry per tool call, per user message, etc.) while history is collapsed to one `[compacted]` summary per completed phase. Use `GET /agent/stream/<sid>` or read the jsonl directly to see what actually happened.
